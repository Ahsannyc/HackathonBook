# Data Model: Better Auth Integration and User Background Questionnaire

## User Entity
- **id**: string (primary key, auto-generated by Better Auth)
- **email**: string (required, unique, validated)
- **name**: string (optional)
- **createdAt**: DateTime (auto-generated)
- **updatedAt**: DateTime (auto-generated)
- **emailVerified**: boolean (default: false)
- **image**: string (optional, for profile picture)

## User Background Entity
- **id**: string (primary key, auto-generated)
- **userId**: string (foreign key to User.id, unique per user)
- **softwareExperience**: string (enum: "beginner", "intermediate", "advanced", "expert")
- **hardwareExperience**: string (enum: "beginner", "intermediate", "advanced", "expert")
- **primaryFocus**: string (enum: "software", "hardware", "both", "theory")
- **backgroundDetails**: string (optional, free text about user's background)
- **createdAt**: DateTime (auto-generated)
- **updatedAt**: DateTime (auto-generated)

## Session Entity (managed by Better Auth)
- **id**: string (primary key, auto-generated)
- **userId**: string (foreign key to User.id)
- **expiresAt**: DateTime (session expiration time)
- **createdAt**: DateTime (auto-generated)
- **updatedAt**: DateTime (auto-generated)

## Relationships
- User (1) → (0 or 1) User Background (one-to-one relationship)
- User (1) → (many) Session (one-to-many relationship)

## Validation Rules
- User email must be properly formatted
- User email must be unique
- User Background userId must reference an existing User
- Only one User Background record per user allowed
- Software and hardware experience values must be from the defined enums

## State Transitions
- User: Created → Email Verified → Active
- Session: Created → Active → Expired
- User Background: Not Collected → Partially Collected → Complete
